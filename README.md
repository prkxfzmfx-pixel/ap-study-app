# 応用情報 学習記録（PWA）

応用情報技術者試験の学習を記録・可視化し、内蔵の過去問で演習できるスマホ向けPWA。GitHub Pagesで配信する。

## 使い方（スマホ / Chrome）

1. ChromeでGitHub PagesのURLを開く
2. メニュー →「ホーム画面に追加」
3. 以降はホーム画面のアイコンから起動（オフラインでも動く）

## 機能

- **試験日カウントダウン**: ホームに「試験まであと N 日」を大きく表示。試験日・試験名は設定で変更可能。
- **学習ログ**: 日付ごとに午前／午後・問題数・正答数・メモを記録。月カレンダーに学習日をマーク、連続学習日数（ストリーク）を表示。
- **正答率グラフ**: 午前・午後別の1日ごとの正答率を折れ線表示。合格ライン60%の水平線つき。手動記録と演習結果を合算。
- **過去問演習（アプリ内完結）**: 収録した過去問をランダム／年度指定／復習モードで出題。ア〜エをタップすると即時正誤判定＋解説を表示し、結果は学習ログ・グラフへ自動連携。間違えた問題は弱点リストに入り、連続2回正解で克服。各問に自分用メモを保存可。
- **設定**: 試験日変更、統計、JSONエクスポート／インポート、演習習熟データのリセット。

## 試験日の既定値と根拠

既定の試験日は **2026-10-28**（試験名「応用情報 前期(科目A)」）。

- 令和8年度（2026年度）から応用情報技術者試験はCBT方式へ移行し、従来の「秋期（10月の一斉試験）」は廃止。
- 代わりに一定期間内で受験日を予約する方式となり、**前期試験 科目A**の実施期間は **2026年10月28日〜11月10日**（科目B は 11月24日〜12月6日）、後期試験 科目A は 2027年2月6日〜19日。
- CBTでは受験者ごとに試験日が異なるため、既定は「前期試験 科目A の実施開始日 2026-10-28」を採用した。自分の予約日に合わせて設定画面で変更してほしい。
- 出典: IPA「令和8年度 応用情報技術者試験等の申込受付期間及び試験実施期間について」。

## 収録している過去問と出典

- **問題・選択肢・正解の出典**: 応用情報技術者試験 午前問題・解答例（独立行政法人情報処理推進機構 IPA）。IPAは試験問題の使用にあたり許諾申請を不要としている（**出典の明記が条件**）。
- **正解データ**: IPA公式「解答例」PDFから機械抽出し、全問がPDFと一致することを `tools/verify.py` で検証済み。
- **解説（explanation）**: AIが独自に書き下ろしたもの。**IPA公式の見解ではなく、誤りを含む可能性がある**。最終確認は公式解答例や過去問道場で行うこと。アプリの結果画面と設定画面下部にもこの旨を明記している。
- **v1収録**: 令和6年秋 午前 のうち、図・表・プログラムを含まない「本文のみで完結する14問」。
  - IPAの「問題冊子」PDFは全ページが画像（テキスト非埋め込み）で、設問文・選択肢を自動抽出できないため、ページ画像を目視して書き起こしている。収録数は今後 `tools/` のパイプラインで拡張する。
- **解答例のみ検証済みの回**（問題本文は未収録）: 令和6年春・令和5年秋・令和5年春・令和4年秋（各80問、`data/answers.json`）。令和4年春はIPAの解答例PDFがテキスト非埋め込みのため除外。

## データの保存場所

記録・習熟データは**端末のブラウザ内（localStorage キー `apstudy.v1`）**にのみ保存され、サーバーには送信されない。
機種変更や別端末との共有は 設定 → エクスポート／インポート（JSON）で行う。

## ファイル構成

| パス | 内容 |
|---|---|
| index.html | アプリ本体（CSS/JS込みの単一ファイル、外部CDN不使用） |
| questions/manifest.js | 収録回の一覧（マニフェスト）。アプリはこれを読んで各回を動的ロード |
| questions/r6a.js | 令和6年秋の問題データ（`AP_REGISTER(meta, questions)`） |
| sw.js | Service Worker（ネットワーク優先・オフラインキャッシュ） |
| manifest.webmanifest / icons/ | PWAメタデータとアイコン |
| data/answers.json | IPA解答例から抽出・検証した正解データ（検証の基準） |
| tools/ | 過去問取り込みパイプラインと検証スクリプト（下記） |

## 過去問の回を追加する（パイプライン）

```
# 1. tools/exams.json に回とPDF URLを追記（a=秋, h=春）
# 2. ワンコマンドでDL→解答抽出→answers.json再生成→問題ページ画像化
python tools/build_exam.py r6h
# 3. data/img/r6h/pNN.png を見て図表なしの問題を questions/r6h.js に書き起こす
#    （AP_REGISTER形式。正解は data/answers.json の値を使う）
# 4. questions/manifest.js の exams に r6h を追加、sw.js の ASSETS とテストも更新
# 5. 検証（正解が公式解答例と全問一致するか等）
python tools/verify.py
```

将来的に過去問道場相当（平成21年春〜）まで、同じ手順で回を足していける構造にしている。

## 更新方法

index.html 等を編集して `git push` するだけでGitHub Pagesに反映される（ネットワーク優先取得なので、オンラインで開き直せば新版になる）。詳細な修正フローは `CLAUDE.md` を参照。
